# Person gRPC Service

Микросервис для получения и хранения информации о людях

## Сущности

```yml
persons:
  - id           # UUIDv7
  - first_name   # string
  - last_name    # string
  - patronymic   # *string
  - age          # uint clamp [1,120]
  - gender       # uint enum (0 - unspecified, 1 - male, 2 - female)
  - nationality  # string

person_emails:
  - id         # big serial
  - person_id  # UUIDv7
  - email      # string

```

## Методы

| Метод          | Описание                                           |
|----------------|----------------------------------------------------|
| `CreatePerson` | Создать человека, обогатить данными из внешних API |
| `GetPerson`    | Получить человека по ID                            |
| `ListPersons`  | Список людей с пагинацией                          |
| `UpdatePerson` | Обновить данные человека (PATCH-логика)            |
| `DeletePerson` | Удалить человека                                   |

## Архитектура

Проект реализован в парадигме гексагональной архитектуры
- `cmd/` - точки входа в приложение (seed базы данных, запуск сервера)
- `internal/domain/` - бизнес-сущности, изолированные от внешних библиотек
- `internal/ports/` - интерфейсы взаимодействия с PostgreSQL, gRPC и внешними HTTP сервисами обогащения данных
- `internal/adapters/` - реализации портов
- `internal/usecase/` - реализация бизнес-логики

## Переменные окружения

В проекте присутствует файл `.env.example`. Он содержится в актуальном состоянии. Можно сделать
```bash
cp .env{.example,}
```
или 
```bash
cp .env.example .env
```
После этого точечно настройте параметры под свои нужды.

### Описание переменных

| Название                         | Описание                                                                                                                                                   | Значение по умолчанию |
|----------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------|
| **DB_ADDRESS**                   | PostgreSQL IP / alias                                                                                                                                      | -                     |
| **DB_PORT**                      | PostgreSQL port                                                                                                                                            | -                     |
| **DB_USER**                      | PostgreSQL username                                                                                                                                        | -                     |
| **DB_PASSWORD**                  | PostgreSQL password                                                                                                                                        | -                     |
| **DB_NAME**                      | PostgreSQL database name                                                                                                                                   | -                     |
| **GRPC_PORT**                    | Порт, который будет слушать сервер                                                                                                                         | **50051**             |
| **EXTERNAL_API_TIMEOUT_SECONDS** | Длительность ожидания ответа от стороннего Enricher API (сек)                                                                                              | **5**                 |
| **EXTERNAL_API_MAX_CONCURRENT**  | Максимальное количество конкурентных горутин в Enricher                                                                                                    | **20**                |
| **SHUTDOWN_TIMEOUT_SECONDS**     | Количество секунд, которые сервер ждет, после получения сигнала на закрытие и отправки команды GracefulStop, прежде чем перейти к принудительному закрытию | **15**                |
| **LOG_LEVEL**                    | Уровень вербальности логов `log/slog`. `[-4]Debug - [0]Info - [4]Warn - [8]Error`                                                                          | **0**                 |
| **ENVIRONMENT**                  | `development` / `production`                                                                                                                               | `development`         |

## Тестирование

Бизнес-логика сервиса покрыта модульными тестами (`internal/usecase/person_test.go`). Тесты покрывают:
* Валидацию некорректных входных данных
* Корнер-кейсы для методов бизнес-логики
* PATCH логику обновления данных

## Устойчивость к отказам

Интеграция с внешними API обогащения данных защищена механизмом повторных попыток и ограничена по конкурентности 
через семафор, что предотвращает DOS внешних сервисов при высоких нагрузках
