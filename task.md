## Контекст

Необходимо реализовать gRPC-сервис для хранения и получения информации о людях. Сервис должен быть готов к высокой нагрузке в production-окружении — это должно отражаться в принятых решениях.

---

## Задача

При создании записи о человеке сервис обращается к внешним HTTP API для обогащения данных:

- возраст — [agify.io](https://agify.io/)
- пол — [genderize.io](https://genderize.io/)
- национальность — [nationalize.io](https://nationalize.io/)

Внешние сервисы имеют ограничения и могут быть временно недоступны.

### Сущность

```
Person:
  - id
  - first_name
  - last_name
  - patronymic     (опционально)
  - age            (из agify.io)
  - gender         (из genderize.io)
  - nationality    (из nationalize.io)
  - emails         (список, может быть пустым)
```

### Методы

|Метод|Описание|
|---|---|
|`CreatePerson`|Создать человека, обогатить данными из внешних API|
|`GetPerson`|Получить человека по ID|
|`ListPersons`|Список людей с пагинацией|
|`UpdatePerson`|Обновить данные человека|
|`DeletePerson`|Удалить человека|

Seed-данные (минимум 10 человек) загружаются отдельной командой, не при старте сервиса.

## Требования

**База данных** Выбор СУБД — на усмотрение кандидата. Схема управляется через миграции. Предоставить `docker-compose` для запуска инфраструктуры.

**Завершение работы** Сервис корректно обрабатывает системные сигналы завершения.

**Логирование** Структурированные логи. Что и когда логировать — на усмотрение кандидата.

**Тесты** Наличие тестов обязательно. Подход — на усмотрение кандидата.
